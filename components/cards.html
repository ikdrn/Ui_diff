6. スケルトンカード（ローディング中）
データ取得中はスケルトンを表示。レイアウトシフトを防ぎ、何かが来ることを示す。 
/* CSS */
@keyframes shimmer {
  0%   { background-position: -200% 0; }
  100% { background-position:  200% 0; }
}
.skeleton {
  background: linear-gradient(90deg,
    var(--surface-2) 25%, var(--border) 50%, var(--surface-2) 75%
  );
  background-size: 200% 100%;
  animation: shimmer 1.4s infinite;
  border-radius: 4px;
}

7. フリップカード
ホバーで裏面が表示される。表: 概要 / 裏: 詳細・アクション という使い方が典型。
📐
デザイン原則
ホバーで詳細を見る
コア原則
  • ✦ 近接性の法則
  • ✦ 視線の流れ
  • ✦ コントラスト
  • ✦ 反復と整合性
/* CSS */
.flip-card-wrapper { perspective: 800px; }
.flip-card-inner {
  position: relative;
  transition: transform 0.5s ease;
  transform-style: preserve-3d;
}
.flip-card-wrapper:hover .flip-card-inner {
  transform: rotateY(180deg);
}
.flip-card-front,
.flip-card-back {
  position: absolute; inset: 0;
  backface-visibility: hidden;
}
.flip-card-back { transform: rotateY(180deg); }

<!-- HTML -->
<div class="flip-card-wrapper" style="width:220px;height:260px;">
  <div class="flip-card-inner">
    <div class="flip-card-front">表面</div>
    <div class="flip-card-back">裏面</div>
  </div>
</div>
