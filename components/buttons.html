Buttons

インタラクションの起点となるUI要素。9種類のバリアント — ソリッド・アウトライン・サイズ・アイコン・ローディング・無効・グループ・全幅・確認パターン。

1. Solid Buttons
いつ使うか: primary = ページ内の主要CTA 1つだけ。success/warning/danger は状態確認ダイアログのアクションボタンに。ghost はキャンセル・補助アクションに。
<button class="btn btn-primary">Primary</button>
<button class="btn btn-secondary">Secondary</button>
<button class="btn btn-success">Success</button>
<button class="btn btn-warning">Warning</button>
<button class="btn btn-danger">Danger</button>
<button class="btn btn-ghost">Ghost</button>
2. Outline Buttons
いつ使うか: Solidより視覚的重量が軽い。同一エリアに複数ボタンがある場合の副次的アクション。白背景カード内で浮かせたくないとき。
<button class="btn btn-outline-primary">Primary</button>
<button class="btn btn-outline-secondary">Secondary</button>
<button class="btn btn-outline-success">Success</button>
<button class="btn btn-outline-warning">Warning</button>
<button class="btn btn-outline-danger">Danger</button>
<button class="btn btn-outline-ghost">Ghost</button>
3. Size Variants
いつ使うか: sm = テーブル・リスト内の補助アクション。md(デフォルト) = フォーム・カード内。lg = セクション内の主要アクション。xl = ヒーローエリアのCTA。
btn-sm
default
btn-lg
btn-xl

Edge case: 長いラベルはwhite-space:nowrapで改行せず切り取る

<button class="btn btn-primary btn-sm">Small</button>
<button class="btn btn-primary">Medium (default)</button>
<button class="btn btn-primary btn-lg">Large</button>
<button class="btn btn-primary btn-xl">Extra Large</button>
4. Icon Buttons
いつ使うか: 左アイコン = 操作の種類を先に伝える（保存・削除など）。右アイコン = 方向や展開を示す（→・↓）。アイコンのみ = ツールバー・テーブルの操作列でスペースを節約。
<!-- 左アイコン -->
<button class="btn btn-primary">
  <svg ...>...</svg>
  Save
</button>

<!-- 右アイコン -->
<button class="btn btn-outline-primary">
  Next
  <svg ...>...</svg>
</button>

<!-- アイコンのみ -->
<button class="btn btn-primary btn-icon-only" title="設定">
  <svg ...>...</svg>
</button>
5. Loading State
いつ使うか: APIリクエスト中・フォーム送信中・ファイルアップロード中。ユーザーに「処理中」と伝えて二重送信を防ぐ。

クリックして3秒間ローディングをシミュレート:

送信完了 
<!-- HTML -->
<button class="btn btn-primary loading">
  <span class="btn-spinner"></span>
  Saving…
</button>

/* CSS */
.btn.loading { pointer-events: none; }
.btn-spinner {
  width: 14px; height: 14px;
  border: 2px solid rgba(255,255,255,0.3);
  border-top-color: #fff;
  border-radius: 50%;
  animation: btn-spin 0.65s linear infinite;
}
@keyframes btn-spin { to { transform: rotate(360deg); } }

// JavaScript: 送信時にローディング状態を付与
btn.classList.add('loading');
btn.innerHTML = '<span class="btn-spinner"></span> Saving…';
// 完了後に解除
btn.classList.remove('loading');
btn.textContent = 'Submit';
6. Disabled State
いつ使うか: 必須フォームが未入力・権限がない操作・前の処理が完了していない場合。disabled はクリック不能だが理由を tooltip や説明文で補足すると UX が向上する。

Edge case: フォーム未入力時は送信ボタンを無効化し、条件を満たすと有効化

<!-- HTML属性で無効化 -->
<button class="btn btn-primary" disabled>Primary disabled</button>

<!-- CSSクラスで無効化 (JS操作向け) -->
<button class="btn btn-outline-primary disabled">Outline disabled</button>

/* CSS */
.btn:disabled,
.btn.disabled {
  opacity: 0.45;
  cursor: not-allowed;
  pointer-events: none;
}
7. Button Group
いつ使うか: テキストエディタのツールバー・表示モード切替（リスト/グリッド）・フィルター選択など、関連する複数の選択肢を横並びで提示する場合。
<div class="btn-group">
  <button class="btn btn-ghost">Left</button>
  <button class="btn btn-ghost">Center</button>
  <button class="btn btn-ghost">Right</button>
</div>

/* CSS */
.btn-group { display: inline-flex; }
.btn-group .btn { border-radius: 0; border-right-width: 0; }
.btn-group .btn:first-child { border-radius: var(--radius) 0 0 var(--radius); }
.btn-group .btn:last-child  { border-radius: 0 var(--radius) var(--radius) 0; border-right-width: 1.5px; }
8. Full Width Button
いつ使うか: モバイルのフォーム送信・カード下部の唯一のCTA・ログインページのSubmitなど、ボタンが領域の幅全体を使うべき場合。
<button class="btn btn-primary btn-block btn-lg">Sign In</button>

/* CSS */
.btn-block {
  width: 100%;
  justify-content: center;
}
9. Danger Confirmation Pattern
いつ使うか: アカウント削除・データ一括消去など「取り消せない操作」の前にワンクッション置く。モーダルを開かずインラインで確認が必要な軽量シナリオに。
Pattern A: ラベル変化（1クリック目→確認→2クリック目→実行）
削除しました
Pattern B: 確認ボタン出現（Cancel で取り消し可能）
本当に削除しますか？
全件削除しました 
<!-- Pattern A: 1クリック目でラベル変化 -->
<button class="btn btn-danger" id="del-btn">Delete</button>

<script>
let pending = false, timer;
document.getElementById('del-btn').addEventListener('click', function() {
  if (!pending) {
    pending = true;
    this.textContent = 'Click again to confirm';
    this.classList.add('confirm-pending');
    timer = setTimeout(() => {
      pending = false;
      this.textContent = 'Delete';
      this.classList.remove('confirm-pending');
    }, 3000);
  } else {
    clearTimeout(timer);
    // 実際の削除処理
    console.log('deleted');
  }
});
</script>
